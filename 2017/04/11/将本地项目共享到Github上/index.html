发表于 2017-04-11

将本地项目源代码托管仓库共享到github上

step 1：本地维护repositories

  • 在项目文件夹目录下打开命令行，输入git init，在本地初始化一个源代码托管仓库 – 隐藏文件夹（.git）；
  • 在项目文件夹下新建一个.gitignore文件，将非源代码文件忽略托管：在.gitignore文件中输入不需要被托管的文件名字，一行一个（可以是文件夹）；
  • 在命令行输入git add .，将托管源代码打包放入仓库；
  • 输入git commit -m 'name(随便取名字)'，给刚刚放入仓库的包贴上标签；

    step 2：在github上new一个repository；

  • 输入git remote add origin 地址(这个地址是github给出的)指定本地库上传到github上哪个库；
  • 输入git push -u origin master将本地库共享到github上；

    实现web页面网络预览

    只能将本地.git文件夹共享到指定地址的gh-pages分支上，才能实现；访问地址是个人github的account+github+io→

    ++joy-lzz.github.io++

    step 3：新建一个gh-pages分支

  • 输入git branch gh-pages，新建一个gh-pages分支；
  • 输入git checkout gh-pages，切换到这个分支；

    将源代码托管到这个分支上

  • 输入git push -u origin gh-pages;

完成上述步骤，就可以实现将本地git的仓库共享到github上供团队使用；同时，还能实现在线预览正在编写的网页！
